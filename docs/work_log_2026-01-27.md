## 1. MCP Graph 쿼리 수정 (Backend)

### 문제
- **Problem Discovery**: `p.text` 속성이 존재하지 않아 빈 결과 반환
- **Tech Clustering**: `t.name` 속성이 존재하지 않아 빈 결과 반환

### 원인 분석
`inspect_schema.py` 스크립트로 Neo4j 스키마 조사:
- `Problem` 노드: `description` 속성 사용 (not `text`)
- `Technology` 노드: `code` 속성 사용 (not `name`)

### 수정 내용
**파일**: `mcp/db/graph.py`

```python
# Before
WHERE toLower(p.text) CONTAINS toLower($keyword)
RETURN p.text as problem

# After  
WHERE toLower(p.description) CONTAINS toLower($keyword)
RETURN p.description as problem
```

```python
# Before
WHERE toLower(t.name) CONTAINS toLower($keyword)
RETURN t.name as technology

# After
WHERE toUpper(t.code) CONTAINS toUpper($keyword)
RETURN t.code as technology_code, t.level as technology_level
```

## 테스트 방법

### Graph Analysis 페이지 테스트
1. 브라우저에서 `http://localhost:3301/graph` 접속
2. 로그인
3. **IPC Classification** 탭 선택
4. Section → Class → Subclass 순서로 선택 (예: G → G02 → G02F)
5. "Search IPC" 클릭
6. 결과 확인:
   - "Interactive Graph View"에 노드/엣지 시각화
   - "Raw Graph Manifest"에 JSON 데이터

### Problem Discovery 테스트
1. **Problem Discovery** 탭 선택
2. 키워드 입력 (예: "heat", "battery")
3. 검색 실행
4. 특허-문제-해결책 그래프 확인
